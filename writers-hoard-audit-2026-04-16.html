Optimization Targets

HIGHExtract shared engine primitives (~700 lines of duplication)
Seven self-contained engines repeat nearly identical patterns: CRUD hooks (~45 lines each), "ensure at least one item" init (~15 lines each), loading spinners (~5 lines × 12), new-item input forms (~25 lines × 6). A shared engine-primitives layer would cut ~700 lines of boilerplate.
Proposed extractions:
→ engines/_shared/useEngineData.ts — generic CRUD hook factory with fetch/add/edit/remove/refresh
→ engines/_shared/LoadingSpinner.tsx — unified spinner component
→ engines/_shared/NewItemForm.tsx — name input + Enter/Escape + create button
→ engines/_shared/useEnsureDefault.ts — auto-create-first-item hook
HIGHMigrate 3 WIP engines to self-contained pattern
Timeline, Writings, and Yarn Board still use the old delegated pattern with centralized hooks in src/hooks/. Their external component files (YarnBoard.tsx at 725 lines, WritingsView.tsx at 555 lines, TimelineView.tsx at 360 lines) should be refactored into engine-local modules to match the target architecture.
Files to migrate:
→ src/hooks/useYarnBoard.ts → src/engines/yarn-board/hooks.ts
→ src/hooks/useWritings.ts → src/engines/writings/hooks.ts
→ src/hooks/useTimelineEvents.ts → src/engines/timeline/hooks.ts
→ src/components/yarn/YarnBoard.tsx (725L) → split into engine components
→ src/components/writings/WritingsView.tsx (555L) → split into engine components
→ src/components/timeline/TimelineView.tsx (360L) → split into engine components
MEDSplit oversized components (5 files > 400 lines)
Several component files exceed healthy size thresholds. The largest — YarnBoard.tsx (725L) and WritingsView.tsx (555L) — combine layout, state logic, and rendering in single files.
YarnBoard.tsx (725L) · WritingsView.tsx (555L) · InspirationGallery.tsx (491L) · AiToolbar.tsx (472L) · ColorPicker.tsx (457L)
MEDRemove or integrate unused shared/hooks/useCRUD.ts
The generic CRUD hook factory at src/shared/hooks/useCRUD.ts was built but is not imported anywhere. Either integrate it as the base for all engine hooks (replacing the duplicated pattern) or remove it as dead code.
src/shared/hooks/useCRUD.ts — 0 imports found
MEDComplete entityResolver.ts cross-engine features
Two TODO stubs remain in entityResolver.ts: resolveEntity() returns null, and searchEntities() returns empty. These block cross-engine linking (e.g., Timeline events referencing Codex entries).
src/shared/utils/entityResolver.ts — lines 69, 81
MED2 components bypass hooks and access DB directly
AiToolbar.tsx and GlobalSearch.tsx import from @/db directly instead of going through domain hooks. This breaks the layered architecture and makes testing harder.
src/components/writings/AiToolbar.tsx · src/components/common/GlobalSearch.tsx
LOWStandardize naming conventions across engines
Hook/function naming varies: useBiographies vs useBrainstormBoards (entity prefix inconsistency), addBiography vs addBoard (short vs full prefix). Loading state derived three different ways. Transaction API usage varies between engines.
LOWStandardize reorder() across all ordered entities
Biography, Dialog/Scene, Storyboard, Video Planner have reorder operations. Diary and Scrapper do not, despite having ordered items. Should be consistent.
LOWAdd Diary engine to project mode presets
Diary engine exists but isn't included in any of the 5 project mode presets (Novelist, Biographer, Reporter, Playwright, Content Creator). Should likely be a default or suggested engine for at least Novelist and Biographer modes.
